# 穿透式工程项目管理驾驶舱 (Penetrative Project Management Dashboard) V0.3

> **基于“价值—资金一体化”管理理念构建的极简、对齐、穿透式项目管理决策驾驶舱原型。**
> 本子项目采用 **Next.js 16 (Webpack 模式) + React 19 + Tailwind CSS + TypeScript** 架构构建，将传统的挣值管理（EVM）与实际资金收付口径在同一张图表内完成极致的对齐与瀑布式穿透下钻。

---

## 🌟 管理理念与核心价值

在传统工程项目管理中，**进度计划（确权 EV）**、**实际成本**与**实际现金收付**往往分处不同报表，口径割裂且时相不一。

V0.3 版本贯彻**“一体化穿透”**方法论，实现了：
1. **统一视域标尺**：以计划合同收入（=100%）作为全局统一水平轴（0 ~ 1000万）。
2. **业主/内部同轴对齐**：总确权回款条与材料、分包、其他成本大类、实际利润大类在同一条水平轴上按预算边界摆放对齐，一眼识别盈亏与融资窗口。
3. **“穿透”下钻审计**：支持从合同总盘、大类分类、供应商预算占比，一直穿透挖掘至单笔最底层的实际收款/付款凭证。
4. **像素级绝对对齐**：摒弃了行级别的缩进设计，标签栏统一采用 `w-72` 固定像素宽度，展开折叠时条形图始终保持在完美的垂直对齐线上。

## 🎨 极简图形引擎 (Minimalist Graphic Engine)

为了提供精致的物理触感与数据高可读性，系统建立了一套组件库 **`PenetrativeGraphicEngine.tsx`**：

*   **底座槽轨道 (`BulletTrack`)**：代表控制预算授权边界，使用柔和浅灰（`#f1f5f9`）与细致内阴影，仿真物理刻入卡槽的凹陷质感。
*   **实际进度柱 (`ActualBar`)**：代表实际确权或实际成本。在预算边界内呈现主色（蓝/靛/紫），一旦发生超支，溢出部分自动切换为脉冲警告红色。
*   **资金液态流 (`CashFlowBar`)**：代表回款收款或付款支出。以高度 60% 的半透明绿色与滚动斜条纹叠放在实际条上方，呈现“资金流向”的液态流动动画。
*   **预算控制卡点 (`BudgetMarker`)**：在各预算限额点画垂直刻度标记。
*   **绝对对准连接树 (`ExpandConnector`)**：在标签栏内部绝对定位（Level 2 位于 `left={16}`，Level 3 位于 `left={32}`），还原严密的折叠树形线条。

## ⚙️ 系统工程特性

### 1. 100% 修复 SSR 水合冲突 (Hydration Mismatch Fix)
*   **技术难点**：在 React 19 + Next.js 环境下，因为仿真引擎随机生成的财务流水数据在 Server 端与 Client 端渲染时产生微弱噪声冲突，容易触发 React 的水合崩溃（Hydration Mismatch），导致 React 所有定时器、折叠交互和浮动条逻辑全部锁死。
*   **极致方案**：在 `app/page.tsx` 入口处对核心驾驶舱实施 **SSR 动态打包隔离** (`dynamic(..., { ssr: false })`)，彻底消除水合死穴，保证在各种网络状况下页面闪电级加载，交互零卡顿。

### 2. 极简浅色画卷模式 (Light Mode 画卷)
*   告别压抑的暗色系，以极简清爽的高对比 slate-50/暖白（`#f8fafc`）作为主画布，纯白轻盈卡片配合三重微弥散阴影层，完美烘托出彩色极简 3D 胶囊的温润与光泽。

### 3. 随动浮动控制条 (Sticky Slider Bar)
*   演练控制面板被设计为 `sticky top-4 z-40` 的高档毛玻璃玻片。随着页面的向下滚动，演练进度条会轻盈地滑行并吸附在视口顶部，极大地方便了管理者的演练点按。

### 4. 绝对垂直对齐
*   通过将左侧标签栏在 HTML 层级彻底隔离缩进，消除了任何行容器边距。这使得图表中所有材料、分包、确权、利润大类以及子级供应商的进度条能够以 `304px` 统一的水平起始像素排列，完全对齐顶部刻度轴。

## 🚀 快速开始

请按照以下指令在本地运行并调试此原型项目：

### 1. 安装依赖
在项目根目录下安装所有 npm 依赖包：
```bash
cd dashboard
npm install
```

### 2. 启动开发服务器 (Webpack 模式)
使用 Webpack 引擎启动开发服务器（这可以有效规避因为 workspace 中包含中文路径可能触发的 Turbopack 编译报错）：
```bash
npm run dev
```

### 3. 生产环境构建验证
打包生成高度优化的静态分发生产包：
```bash
npm run build
```
